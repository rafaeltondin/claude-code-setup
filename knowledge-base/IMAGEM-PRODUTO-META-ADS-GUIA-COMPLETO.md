# Imagem de Produto em Criativos Estaticos para Meta Ads
## Guia Tecnico Completo de Boas Praticas

> **Objetivo:** Documentacao tecnica para orientar a criacao de criativos estaticos (Facebook/Instagram) com foco no uso otimizado de imagens de produto.
> **Baseado em:** Pesquisas de eye-tracking, benchmarks Meta Ads 2025-2026, diretrizes oficiais Meta, e boas praticas da industria.

---

## 1. IMAGE-TO-TEXT RATIO

### Regra da Meta (historico)
- A Meta **removeu a regra rigida dos 20%** de texto em 2020
- Porem, o **algoritmo ainda penaliza** imagens com muito texto: menor alcance e custo maior
- A penalizacao e gradual: quanto mais texto, menor a entrega

### Proporacao Ideal para Conversao

| Faixa de Texto | Impacto no Alcance | Recomendacao |
|---|---|---|
| **0% (sem texto)** | Maximo alcance | Ideal para awareness e produtos visuais |
| **1-10%** | Alcance otimo | **Sweet spot para conversao** |
| **10-15%** | Leve reducao | Aceitavel com CTA curto |
| **15-20%** | Reducao moderada | Limite maximo recomendado |
| **>20%** | Reducao significativa | Evitar — CPC e CPM sobem |

### Dados de Performance
- Ads com imagens otimizadas alcancam **8.95% de taxa de conversao** media (2025)
- ROAS de **155%** em campanhas com imagens otimizadas vs nao-otimizadas
- Vantagem de **$5.13 no CPM** para imagens bem formatadas
- **Muitas marcas de e-commerce** reportam que ads com **zero texto overlay** performam melhor

### Recomendacao Pratica
```
REGRA: Manter texto overlay abaixo de 15% da area da imagem.
IDEAL: 5-10% — apenas headline principal ou badge de preco.
TESTE: Sempre testar versao sem texto vs com texto.
```

## 2. HERO IMAGE PLACEMENT (Posicionamento da Foto do Produto)

### Principio Geral
- O produto deve ser o **primeiro elemento que o olho captura** (ponto focal primario)
- Eye-tracking confirma: elementos maiores e com alto contraste capturam atencao primeiro
- Posicionar o produto onde o olho naturalmente comeca a escanear (canto superior esquerdo em culturas ocidentais)

### Posicionamento por Formato

| Formato | Ratio | Posicao do Produto | Justificativa |
|---|---|---|---|
| **Feed Vertical** | 4:5 | **Topo (40-50% superior)** | Olho comeca no topo; info abaixo |
| **Stories/Reels** | 9:16 | **Terco superior** | Safe zone; swipe-up/CTA fica embaixo |
| **Quadrado** | 1:1 | **Topo ou centro-esquerda** | Z-pattern: olho comeca top-left |
| **Landscape** | 16:9 | **Esquerda (40-50%)** | Leitura esquerda-direita; info a direita |
| **Wide** | 1.91:1 | **Esquerda compacta (35-40%)** | Espaco horizontal limitado |

### Regras de Ouro
1. **Nunca centralizar o produto no centro exato** — use regra dos tercos (interseccoes)
2. **Produto no topo/esquerda** = escaneamento natural (Z-pattern ocidental)
3. **Rostos direcionando o olhar para o produto** aumentam fixacao (eye-tracking comprovado)
4. **Fundo contrastante** — produto deve se destacar do background (alto contraste)

### Hierarquia Visual Recomendada
```
1o. Produto (hero image) — captura atencao
2o. Headline / Beneficio — comunica valor
3o. Badge / Preco — cria urgencia
4o. CTA — direciona acao
```

## 3. REGRA DE OCUPACAO DA IMAGEM (40-60%)

### Por que o Produto Deve Ocupar 40-60% do Criativo

#### Dados e Fundamentos
- **Golden Ratio (1.618:1):** A proporcao aurea sugere que o hero image ocupe **~61.8% da largura**, com texto/whitespace nos 38.2% restantes
- **Regra 60-30-10 de cor:** 60% cor dominante (fundo), 30% secundaria (imagem/graficos), 10% accent (CTA)
- **3-Element Rule:** Limitar a 3 pontos focais (headline, produto, CTA) — cada um recebe ~33% da atencao. Com 12+ elementos, cada um recebe apenas 8-12%

#### Eye-Tracking: O que os Estudos Mostram
- Viewers gastam **~5 segundos** antes de rolar (5-Second Rule)
- Imagens grandes e contrastantes recebem **fixacao primaria** (primeiro ponto de olhar)
- O que designers acham que o viewer ve primeiro **frequentemente nao e** o que realmente ve
- Heatmaps revelam que CTAs bonitos mas mal posicionados recebem **zero atencao**

#### Impacto no CTR e Conversao

| Ocupacao da Imagem | Resultado |
|---|---|
| **< 30%** | Produto perde impacto; nao captura atencao no feed |
| **30-40%** | Aceitavel para ads com muito texto (testimonials) |
| **40-60%** | **Faixa ideal** — equilibrio entre impacto visual e espaco para info |
| **> 60%** | Bom para awareness puro, mas pouco espaco para CTA/beneficios |
| **> 80%** | Produto domina, mas sem contexto/CTA — baixa conversao |

#### Recomendacao por Objetivo

| Objetivo | Ocupacao Ideal | Motivo |
|---|---|---|
| Awareness | 50-70% | Maximo impacto visual |
| Consideration | 40-55% | Equilibrio produto + beneficios |
| Conversion | 35-50% | Espaco para preco, CTA, urgencia |
| Retargeting | 45-60% | Produto reconhecivel + oferta |

## 4. TECNICAS DE COMPOSICAO VISUAL PARA ADS

### 4.1 Regra dos Tercos
```
Dividir o criativo em grade 3x3:
+-------+-------+-------+
|       |       |       |
|   *   |       |   *   |  <- interseccoes = pontos focais
|       |       |       |
+-------+-------+-------+
|       |       |       |
|       |       |       |
|       |       |       |
+-------+-------+-------+
|       |       |       |
|   *   |       |   *   |  <- produto ou CTA aqui
|       |       |       |
+-------+-------+-------+
```
- Posicionar produto em uma das **4 interseccoes** — nunca no centro morto
- Cria composicao **dinamica e interessante** visualmente
- Nike, Apple e grandes marcas usam sistematicamente

### 4.2 Z-Pattern (Layouts Visuais / Menos Texto)
```
INICIO ──────────────────> (Logo / Badge)
   ↓                         ↓
    ↘                       ↗
      ↘  (Produto aqui)  ↗
        ↘               ↗
   ↓                         ↓
(Info secundaria) ──────> (CTA)
```
- **Top-left:** Logo ou badge de marca
- **Top-right:** Headline principal
- **Centro (diagonal):** Hero image do produto
- **Bottom-left:** Beneficio/preco
- **Bottom-right:** CTA (botao)
- **Ideal para:** Ads visuais com pouco texto, imagens de produto

### 4.3 F-Pattern (Layouts com Mais Texto)
```
████████████████████████  <- Headline (leitura horizontal)
████████████████          <- Sub-headline (mais curto)
██████████                <- Body text (escaneamento vertical esquerdo)
██████
████
     [CTA]
```
- Viewer le horizontalmente no topo, depois escaneia verticalmente pela esquerda
- **Ideal para:** Ads com lista de beneficios, comparacoes, testimonials
- Headline recebe mais atencao, primeiras palavras de cada linha importam

### 4.4 Ponto Focal e Hierarquia Visual
- **Tamanho:** Elementos maiores = mais importancia percebida
- **Contraste:** Alto contraste com fundo = mais atencao
- **Cor:** Cores quentes (vermelho, laranja) avancam; frias (azul) recuam
- **Isolamento:** Produto com whitespace ao redor = mais destaque
- **Direcionalidade:** Setas, linhas, olhar de modelos guiam o olho

### 4.5 Gradiente de Transicao (Fade)
Tecnica para criar transicao suave entre a zona de imagem e a zona de conteudo:

```css
/* Fade vertical: imagem no topo transitando para fundo escuro */
.image-zone::after {
  content: '';
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
  height: 80px;
  background: linear-gradient(
    to bottom,
    transparent 0%,
    rgba(10, 10, 10, 0.6) 40%,
    rgba(10, 10, 10, 1) 100%
  );
  pointer-events: none;
}

/* Fade horizontal: imagem na esquerda transitando para conteudo */
.image-zone::after {
  content: '';
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  width: 100px;
  background: linear-gradient(
    to right,
    transparent 0%,
    rgba(10, 10, 10, 0.6) 40%,
    rgba(10, 10, 10, 1) 100%
  );
}
```

**Beneficios:**
- Elimina corte brusco entre imagem e fundo
- Cria profundidade e sofisticacao visual
- Permite que badges/texto fiquem legiveis na borda da imagem

### 4.6 Overlay de Badges e Precos

```css
/* Badge de desconto sobre a imagem */
.badge-overlay {
  position: absolute;
  top: 12px;
  right: 12px;
  background: rgba(215, 31, 63, 0.92);  /* semi-transparente */
  color: white;
  padding: 6px 14px;
  border-radius: 4px;
  font-weight: 800;
  font-size: 14px;
  text-transform: uppercase;
  letter-spacing: 0.5px;
  backdrop-filter: blur(4px);           /* efeito vidro */
  box-shadow: 0 2px 8px rgba(0,0,0,0.3);
}

/* Badge com sombra de texto para legibilidade */
.price-badge {
  text-shadow: 0 1px 3px rgba(0,0,0,0.5);
}
```

**Posicoes recomendadas para badges:**

| Posicao | Uso Ideal | Cuidado |
|---|---|---|
| **Top-right** | Desconto, "NOVO" | Mais visivel, padrao do mercado |
| **Top-left** | Logo, selo de qualidade | Nao competir com badge principal |
| **Bottom-left** | Preco, parcelas | Verificar safe zone (Stories) |
| **Bottom-right** | CTA inline, "Saiba mais" | Nao sobrepor CTA principal |

## 5. OBJECT-FIT E ENQUADRAMENTO

### Quando Usar Cover vs Contain

| Propriedade | Comportamento | Quando Usar |
|---|---|---|
| `object-fit: cover` | Preenche container, **corta** excesso | Hero images, banners, thumbnails — sem espacos vazios |
| `object-fit: contain` | Mostra imagem **inteira**, pode ter espacos | Produto inteiro precisa aparecer (logo, diagrama) |
| `object-fit: scale-down` | Menor entre contain e natural | Imagens pequenas que nao devem esticar |

### Implementacao para Criativos

```css
/* COVER: Hero image preenche a zona — ideal para lifestyle/ambientacao */
.hero-image {
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: center 30%;  /* produto no terco superior */
}

/* CONTAIN: Produto inteiro visivel — ideal para pack shots */
.product-image {
  width: 100%;
  height: 100%;
  object-fit: contain;
  object-position: center center;
  padding: 10px;  /* respiro ao redor */
}
```

### Object-Position para Centralizar o Produto

```css
/* Produto no centro-superior (mais comum) */
object-position: center 25%;

/* Produto levemente a esquerda (composicao assimetrica) */
object-position: 40% center;

/* Produto no topo (vertical com info embaixo) */
object-position: center top;

/* Produto centralizado com bias para cima */
object-position: 50% 35%;
```

### Manter o Produto Visivel no Crop
1. **Usar `object-position`** para direcionar o ponto focal
2. **Testar em todos os formatos** — o crop muda entre 4:5, 1:1, 9:16
3. **Safe margin de 10-15%** ao redor do produto na foto original
4. **Centralizar o produto na foto source** — facilita qualquer crop
5. **Evitar produto nas bordas** — risco de corte em qualquer aspecto

**Regras:**
- Produto na **esquerda (40-50%)**
- Info e CTA na **direita (50-60%)**
- Gradiente fade **horizontal** (left-to-right)
- Cuidado: formato menos imersivo no mobile
- **Resolucao:** 1920x1080px

## 8. ERROS COMUNS COM IMAGEM EM CRIATIVOS

### 8.1 Imagem Muito Pequena
- **Problema:** Produto nao e identificavel no thumbnail do feed
- **Impacto:** Baixo CTR, viewer nao entende o que esta vendendo
- **Solucao:** Produto deve ocupar **minimo 35%** do criativo
- **Teste:** Se o produto nao e reconhecivel em **50x50px** no thumbnail, esta pequeno demais

### 8.2 Imagem Cortada no Lugar Errado
- **Problema:** `object-fit: cover` corta parte essencial do produto
- **Impacto:** Produto irreconhecivel ou aparencia amadora
- **Solucao:** Usar `object-position` para direcionar o ponto focal
- **Prevencao:** Fotografar com **safe margin de 15-20%** ao redor do produto

### 8.3 Sem Gradiente de Transicao (Corte Brusco)
- **Problema:** Transicao abrupta entre imagem e area de texto
- **Impacto:** Aparencia pouco profissional, layout "colado"
- **Solucao:** Gradiente fade de 60-100px na borda da imagem
- **CSS:** `linear-gradient(to bottom, transparent, rgba(bg-color, 1))`

### 8.4 Badges Ilegiveis sobre Imagem Clara
- **Problema:** Badge de preco/desconto se perde em imagem de fundo claro
- **Impacto:** Informacao de preco invisivel = perda de conversao
- **Solucoes:**
  - `backdrop-filter: blur(6px)` para efeito vidro
  - Fundo semi-transparente escuro (`rgba(0,0,0,0.75)`)
  - Borda sutil (`border: 2px solid rgba(255,255,255,0.3)`)
  - `text-shadow` para contraste
  - `box-shadow` para elevar o badge

### 8.5 Muito Texto Sobre a Imagem
- **Problema:** Texto competindo com o produto pela atencao
- **Impacto:** Penalizacao do algoritmo Meta + confusao visual
- **Solucao:** Texto na zona de conteudo, nao sobre a imagem. Maximo: 1 badge sobre imagem.

### 8.6 Imagem de Baixa Resolucao
- **Problema:** Pixelacao visivel, especialmente em telas Retina
- **Impacto:** Percepcao de baixa qualidade da marca/produto
- **Solucao:** Resolucao minima 2x do tamanho de exibicao
- **Recomendado:** 1440x1800 para 4:5, 1440x1440 para 1:1

### 8.7 Fundo que Compete com o Produto
- **Problema:** Background muito colorido ou detalhado ofusca o produto
- **Impacto:** "Efeito vampiro" — fundo rouba atencao do produto
- **Solucao:** Fundos solidos escuros ou gradientes sutis. Produto em alto contraste.

### 8.8 Nao Testar em Mobile
- **Problema:** Criativo fica bom no desktop mas pequeno/ilegivel no mobile
- **Impacto:** **94-98%** do trafego Meta e mobile — desktop e irrelevante
- **Solucao:** Sempre projetar mobile-first. Testar em tela de 375px de largura.

### 8.9 Foto do Produto Nao Corresponde ao Produto Real
- **Problema:** Imagem generica ou stock photo nao relacionada
- **Impacto:** Confusao, baixa confianca, high bounce rate
- **Solucao:** Usar foto real do produto. Lifestyle shots > stock photos.

### 8.10 Ad Fatigue por Nao Rotacionar Imagens
- **Problema:** Mesma imagem por semanas, frequencia > 3.0
- **Impacto:** CTR cai, CPA sobe, viewer ignora o ad
- **Solucao:** Rotacionar criativos a cada **7-14 dias**. Trocar cor, angulo, ou composicao.
- **Trigger:** Rotacionar quando CTR < 1.5% OU CPA sobe 25%+ OU frequencia > 3.0

### 2026-03-18 - Análise de pixels para posicionamento de texto sobre imagem de produto

**Regra:** Para qualquer criativo com imagem de produto como background, SEMPRE rodar análise de pixels Python ANTES de posicionar texto. Salvar `placement_data.json` com safe zones. Usar `max-height + overflow:hidden` no CSS para garantir que o texto nunca invada a zona do produto.

### 2026-03-18 - Gradiente overlay para legibilidade de texto sobre foto clara
**Contexto:** Texto branco sobre foto de produto com fundo cinza claro — baixo contraste.
**Erro:** Na primeira versão (v1), texto ficou diretamente sobre fundo escuro separado (produto pequeno). Na v2, texto sobre foto sem gradiente era difícil de ler.
**Solução:** Gradiente CSS adaptativo baseado na posição do texto:
